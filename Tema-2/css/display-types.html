Display: inline vs block vs inline-block

1. display: inline (como <span>)

Texto antes

INLINE 1 INLINE 2 INLINE 3

Texto después

Características:
✅ Se coloca en la misma línea
❌ NO respeta width/height
❌ NO acepta margin-top/bottom
✅ Acepta padding-left/right
✅ Se comporta como texto

2. display: block (como <div>)

Texto antes

BLOCK 1
BLOCK 2
BLOCK 3

Texto después

Características:
✅ Respeta width/height
✅ Acepta todos los margins/paddings
❌ Siempre empieza en nueva línea
❌ Ocupa todo el ancho disponible
✅ Se puede centrar con margin: auto

3. display: inline-block (¡LO MEJOR DE AMBOS!)

Texto antes

INLINE-BLOCK 1 INLINE-BLOCK 2 INLINE-BLOCK 3

Texto después

Características:
✅ Se coloca en la misma línea (como inline)
✅ Respeta width/height (como block)
✅ Acepta todos los margins/paddings (como block)
✅ Se puede usar para layouts horizontales
✅ Perfecto para botones, etiquetas, elementos navegación
